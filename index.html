Quick idea (like a real-life story)

Imagine you order something on Amazon. The order has an ID, an amount, and an address. Inside that order, there's delivery status (packed, shipped, out for delivery). The delivery status only makes sense inside the order — that's exactly why we use an inner class. A anonymous inner class is like a sticky note you write once for quick use (no need to create a whole new book/class). An abstract class is a half-finished plan — and combining it with anonymous means we finish the plan right away without creating a named child.

1) Inner Class — simple example

What it is (5th-grade style): A class inside another class. Use it when the inner thing only belongs to the outer thing (like delivery status belongs to an order).

// InnerClassExample.java
public class InnerClassExample {
    private int orderId;
    private int amount;

    public InnerClassExample(int orderId, int amount) {
        this.orderId = orderId;
        this.amount = amount;
    }

    // Inner class (DeliveryStatus)
    class DeliveryStatus {
        private String status;
        public DeliveryStatus(String status) {
            this.status = status;
        }
        public void showStatus() {
            System.out.println("Order ID: " + orderId);
            System.out.println("Amount: " + amount);
            System.out.println("Status: " + status);
        }
    }

    public static void main(String[] args) {
        InnerClassExample order = new InnerClassExample(101, 999);
        InnerClassExample.DeliveryStatus ds = order.new DeliveryStatus("Out for delivery");
        ds.showStatus();
    }
}
Time complexity: O(1) — the program does a fixed number of steps (create objects, print), so time doesn't grow with input size.
Space complexity: O(1) — only a few variables and objects are created regardless of input size.
Why: we only make one order and one DeliveryStatus. If you made N orders the time/space would grow with N, but this example shows a single fixed example.

Real-life usage: Encapsulating data that belongs only to a single parent object. Company example: Amazon-like systems use inner classes or similar patterns to model order + delivery details.

2) Anonymous Inner Class — quick sticky note

What it is (5th-grade style): A tiny class without a name used one time only — like a quick sticky note you throw away after reading.

// AnonymousInnerClassExample.java
public interface Notes {
    void remember();
}

public class AnonymousInnerClassExample {
    public static void main(String[] args) {
        // Create an anonymous inner class that implements Notes
        Notes stickyNote = new Notes() {
            @Override
            public void remember() {
                System.out.println("Remember this formula: a² + b² = c²");
            }
        };

        stickyNote.remember();
    }
}
Time complexity: O(1) — the example runs a fixed action (create and call) once.
Space complexity: O(1) — only one anonymous object is created.
Why: The code creates a single anonymous object and calls one method — constant steps and constant memory.

Real-life usage: When you need a one-off behavior (like a one-time confirmation dialog). Company example: Quick UI callbacks or listeners (many companies with Java UI code — e.g., older Java desktop apps or some backend callback code).

3) Abstract class + Anonymous implementation

What it is (5th-grade style): An abstract class is a half-finished plan. Using an anonymous class means you finish the plan immediately without naming the child — like assigning a short task to someone and not writing their name on a big roster.

// AbstractAnonymousExample.java
abstract class AbstractDelivery {
    abstract void deliver();
}

public class AbstractAnonymousExample {
    public static void main(String[] args) {
        // Anonymous implementation of abstract class
        AbstractDelivery delivery = new AbstractDelivery() {
            @Override
            void deliver() {
                System.out.println("Your order will be delivered by bike.");
            }
        };

        delivery.deliver();
    }
}
Time complexity: O(1) — the example performs a fixed action once.
Space complexity: O(1) — one anonymous object is allocated.
Why: Only one object and one method call exist in the example, so both time and memory stay constant.

Real-life usage: Systems that define a general contract (abstract) and sometimes provide quick one-off implementations (anonymous). Company example: Delivery assignment systems in e-commerce where different deliverers (strategies) are used without making many separate classes.

Short tips & when to use which

  • Inner class: Use when the inner thing belongs ONLY to the outer thing (encapsulation).
  • Anonymous class: Use for quick, one-time tasks — UI callbacks, small listeners, one-off implementations.
  • Abstract + anonymous: When you have a general plan but want to implement it quickly without creating a named subclass.
